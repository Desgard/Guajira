不再简书

从即日起个人简书不再同步 blog 的技术文章。

第一次看见简书是在 2014 年的时候，当时业界天才、知名“写手”王垠开始使用简书写作，当我点开简书主页的时候，觉得很清新、干净。于是我也注册了账号，开始了简书的写作之路。

当时，自己还在做前端，就写了几篇 jQuery 的学习笔记。然后发觉简书上的技术文比较少，还是将个人随笔发上去吧。于是陆续的在上面发布了自己的散文。但是久而久之，也就放弃了这个平台。一些个人的体会：

  • 写作平台的作者自由注册，所以水平参差不一；
  • 很多个人随笔不太愿意公之于众，希望只存在于我的好友圈中。简述的平台当初还没有多少好友，所以很多文章不会有人阅到；

随后的很长时间，我忍受着 Qzone 无法使用 markdown（当然现在也不可以）的痛苦，在 Qzone 这个自由创作平台完成了一篇诗集（之前是送给某人的），很多篇关于生活随想的随笔，以及个人定期的一些总结。写作的生活是很有趣的，因为当你不愿意和别人分享一些苦楚的时候，可以对着文字倾诉。当你开心的时候，你能把喜悦的心情用美好的文字记录下来，成为时间中深深的烙印，每逢回顾，更加温暖。

所以我在 coding 的同时也会多读些散文、小说来充实自己，然后在写点随笔、读后感，觉得自己的观点也有一席之地。😄

在今年 3 月份因为开始进入 iOS 这个大坑，被好友的邀请下又回到了简书，开始发技术文。但是发现简书的 markdown 引擎毫无更新，个人有一些需求无法满足：

  • 不支持 Objective-C 的代码高亮；
  • 不支持 LaTeX；
  • 不支持 Diagram 流程图绘制；

所以在使用了近三个月之后，决定不再与 blog 的技术文同步。只在 blog 与 github 进行文章同步。所以，感谢很多简书关注我的人，给予我长期的支持。

最后也希望简书能够越办越好。虽然不再用其写文，但也陪伴了我两年的大学生活。
